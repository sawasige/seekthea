# Seekthea 改善・新機能提案

> 2026-05-08 / 専門家ラウンドテーブルでの提案 → 1 件ずつ精査して反映

## 現状の強み・弱み

| 観点 | 状態 |
|------|------|
| **強み** | Apple Intelligence ネイティブ要約・分類／2層パーソナライズ（keyword 40% + semantic 60%）／3モード閲覧（Reader/AI/Web）／自動ソース発見／CloudKit 同期／マルチプラットフォーム |
| **弱み** | 検索・通知・ウィジェット・Watch・Share Extension・TTS・記事重複検出・明示的 Dislike・Spotlight/Shortcuts いずれも未実装／VoiceOver 最小限 |

ユーザー本人が Smart News／Yahoo!ニュース派で「RSS リーダーは好きじゃない」前提。
ポジショニングは「**Apple News 的品質感を、日本のニュースに対して、無料で提供する個人開発アプリ**」。

# 採用機能と設計

## D. 興味なしフィードバック（採用 / v1.4〜v1.5）

### 設計方針
**暗黙シグナル中心 + トピック単位救済**。記事ごとに「興味なし」操作させない。

### 仕組み

**既存（そのまま継続）**
- 既読・お気に入りからのキーワード学習（learned keywords）
- impressionCount による表示回数ペナルティ
- 手動 `UserInterest`（手で追加した topic、weight 0.1〜2.0）

**新規追加**
1. **負方向の暗黙学習**：表示されたが開かれない記事のキーワードを negative weight で集計
2. **学習結果の可視化**：内部計算されてる学習結果を、興味画面に「AI が学習中」セクションとして表示（編集不可、情報表示のみ）
3. **救済の動線**：
   - 学習結果カードに「興味トピックに追加」ボタン → 手動 `UserInterest` に昇格
   - 学習結果カードに「除外」ボタン → 別管理の除外キーワード DB に追加
   - 「学習をリセット」で全部やり直し

### 概念整理（重要）
| 種類 | データ | 編集 |
|------|--------|------|
| **手動トピック** (`UserInterest`) | 永続化 | スライダーで重み調整可 |
| **学習結果**（内部計算） | 一時的、計算結果のみ | 編集不可、見るだけ |
| **除外キーワード** | 永続化 | 削除のみ |

スライドできるのは**手動側だけ**。学習側は触らない（触れたら「自動」じゃなくなる）。

### 救済の動線
- 「これを増やしたい」 → 学習カード →「興味トピックに追加」→ 手動側で調整
- 「これを減らしたい」 → 学習カード →「除外」→ 除外リストへ

### 画面イメージ
```
[興味]                                + 追加

  ─ 手動で設定したトピック ─
  プログラミング    ━━●━━━━━
  AI               ━━━●━━━━

  ─ AI が学習中（読書履歴から） ─    [リセット]
  Vision Pro     +0.8   [追加] [除外]
  WWDC           +0.6   [追加] [除外]

  ─ 除外キーワード ─
  芸能ゴシップ                       ✕
  競馬                              ✕
```

## I. 記事詳細：前後記事カード + モード Picker（採用 / v1.5）

### 設計方針
Apple News 的品質感のナビゲーション + 「次が何か見えないと開かない」を解決。

### 仕組み
- **モード切替（Reader / AI / Web）は Segmented Picker** に降格
- **記事末尾近くまでスクロールすると、前後記事カードが fade-in**
  - 左下：前の記事カード（画像 + タイトル + ソース・時刻）
  - 右下：次の記事カード
  - タップで遷移、横スワイプジェスチャも併用可
- 「次の記事」= 開いた時のフィードのリスト順
- セッション中はスコア再計算しない固定スナップショット（既読化で順序が動かないように）

### 残課題（実装時に詰める）
- カードの正確な fade-in トリガー（スクロール 80%? 末尾到達?）
- モード Picker の位置（toolbar 直下 vs 末尾）
- visionOS / macOS のフォールバック

## K. キーボードショートカット最小セット（採用 / v1.5）

### 仕組み
Mac / iPad で物理キーボード接続時の操作性向上。

| キー | 動作 |
|------|------|
| ↓ / ↑ または j / k | 次/前の記事 |
| Space | 開く |
| ⌘R | 更新 |
| s または f | お気に入りトグル |

### 効果
- L（OPML）と組み合わせて Reeder / NetNewsWire 経験者の乗り換え訴求
- Mac / iPad の品質感向上

---

## L. OPML import/export（採用 / v1.5、K とセット）

### 仕組み
- 設定 → インポート/エクスポートに「OPML」追加
- import：DocumentPicker で OPML 選択 → ソース一覧プレビュー → 一括追加（DataDeduplicator で重複排除）
- export：全ソース or カテゴリ別 or 有効ソースのみ

### 効果
- Reeder / NetNewsWire / Feedly / Inoreader からの乗り換え障壁ほぼゼロ
- App Store 説明文に「他の RSS リーダーから OPML で簡単移行」追加可能
- 既存ユーザーの設定バックアップにも有用

## M. Spotlight + Siri Shortcuts（採用 / v1.6 以降）

### M-1. Spotlight 検索対応
- お気に入り記事を `CSSearchableItem` で index 登録
- ホーム画面下スワイプ → 過去のお気に入り検索可能
- 軽量実装（お気に入り限定で数十〜百件規模）

### M-2. Siri Shortcuts / App Intents
- 「Hey Siri, Seekthea で今日のニュース」等のフレーズ対応
- ショートカットアプリで Seekthea アクションを組み込める
- F のダイジェスト機能と連携すると本領発揮（朝のルーティンで読み上げ等）

---

## P. TTS 読み上げ（採用 / 品質検証先行 / v1.6 以降）

### 仕組み
- AI 要約の連続再生（記事キュー、Podcast 的体験）
- iOS 26 標準音声（AVSpeechSynthesizer）使用、外部 API なし
- ロック画面・コントロールセンターから再生制御
- バックグラウンド再生対応

### 検証先行
- iOS 26 標準音声で日本語ニュースを読み上げて品質評価
- 単語誤読（NHK / BMW / 人名）の頻度確認
- 使えるレベルなら本実装、ダメなら見送り

### 差別化
- 日本のニュースアプリで TTS 対応はほぼ無い
- Apple News の「News Narrated」は News+ 加入限定 → 無料で代替

# 検証先行タスク（実装前にプロトタイプ）

## F. ダイジェスト通知（BG 動作検証）

### 現状の課題
- BG でスコア計算するには新着記事の AI キーワード抽出が必要
- Foundation Models が BGAppRefreshTask で動くか公式ドキュメントが曖昧

### 検証内容
- 簡単な BGAppRefreshTask + Foundation Models 呼び出しの試作
- 数日間 OS が実行してくれるか観察
- 結果次第で設計確定：
  - 動く → リッチダイジェスト可、Phase 2 路線
  - 動かない → 件数集計通知のみ（「12 件の新着、テクノロジー多め」型）

## G. 「今朝のサマリー」コラム生成

### 検証内容
- 過去 24 時間の Top 10 記事を Foundation Models に投入
- 200 字の総括コラムを実際に生成
- 出力品質を見て採用判断
- ハルシネーション・断定表現のリスク評価

### 採用判断基準
- 読みごたえがあれば採用、F のフォアグラウンドダイジェストの 1 表現として
- 機械的列挙にしかならないなら不採用

## P. TTS 読み上げ品質

### 検証内容
- iOS 26 標準音声で日本語ニュース記事を 5〜10 件読ませる
- 単語誤読、抑揚、聞き取りやすさを評価
- 使えるレベルか判断

## C. ウィジェット（F の評価後）

### 評価方針
- 単独採用は見送り（離脱ユーザーは結局ウィジェットも登録しない）
- F のダイジェスト通知が成立した時点で「セット施策」として再評価
- ASO スクショ用と割り切るなら別途検討

# 不採用

| # | 機能 | 理由 |
|---|------|------|
| A | アクセシビリティ底上げ | スキップ（必要時に再着手） |
| B | FeedFetcher 並列化 | 直列でも速度問題なし（URLCache ヒット効果大） |
| E | 重複記事クラスタ化 | 「URL違う同じ記事」の前提が曖昧、ユーザーニーズなし |
| H | Share Extension | Smart News / Yahoo で完結している、外部記事保存ニーズ低い |
| J | スワイプジェスチャアクション | カードからお気に入りする動線需要が薄い、カテゴリ切替と衝突 |

# ポジショニング

> **「Apple News 的品質感を、日本のニュースに対して、無料で提供する個人開発アプリ」**

- Smart News / Yahoo!ニュース：コンビニ感、賑やか、大衆向け
- Reeder：硬派 RSS リーダー、玄人向け
- Apple News：上質・編集キュレーション・知識層向け（日本未提供）
- **Seekthea**：上質感 + AI パーソナライズ + iOS 上質志向層向け
