# Apple Intelligenceで動く無料ニュースアプリを作った — Seektheaの設計と実装

## はじめに

RSSリーダーは好きだけど、フィードが増えると未読が溜まって結局読まなくなる。かといって、キュレーションアプリは「誰かのおすすめ」であって「自分のおすすめ」ではない。

「自分が選んだソースから、AIが要約して、自分の興味に合わせて並べてくれるアプリ」が欲しくて、Seektheaを作りました。

特徴はシンプルで、**サーバーなし、APIキーなし、サブスクなし**。Apple Intelligenceのオンデバイス処理だけで完結するので、月額コストはゼロです。

この記事では、アプリの紹介と、Apple Intelligence（Foundation Modelsフレームワーク）をどう使っているかを書きます。

## Seektheaとは

**Seekthea**（シークシア）は、ニュースサイト・テック系メディア・ソーシャルメディアなど100以上のRSSソースからトレンドを収集し、Apple Intelligenceで要約・分類するアプリです。

名前はSeek（探す）+ Thea（ギリシャ語で「眺め・視界」、ギリシャ神話の視覚の女神テイアの名前でもある）から。ウェブ全体のトレンドを見通して探し出す、という意味を込めています。

### できること

- **AI要約**: 記事をデバイス上で要約。タイムラインを眺めるだけで要点がわかる
- **カテゴリ自動分類**: テクノロジー、ビジネス、エンタメなど、AIが記事を自動でカテゴリに振り分ける
- **パーソナライズ**: 読んだ記事やお気に入りから興味を学習し、「おすすめ」の精度が上がっていく
- **ソース自動発見**: Google Newsのトレンドから新しいソースを見つけて提案
- **iCloud同期**: iPhone / iPad / Mac / Vision Proで既読・お気に入り・ソース設定が同期
- **完全無料**: アカウント登録も広告もなし

### 対応プラットフォーム

iPhone / iPad / Mac / Apple Vision Proのマルチプラットフォーム対応です。SwiftUIで1つのコードベースから全プラットフォームに対応しています。

## アーキテクチャ — サーバーを使わない設計

Seektheaの設計で最もこだわったのは「**外部サーバーに一切依存しない**」ことです。

```
[RSSフィード] → FeedKit → OGP補完 → Apple Intelligence → SwiftData → SwiftUI
                                        (オンデバイス)      (+ CloudKit同期)
```

| レイヤー | 技術 | コスト |
|----------|------|--------|
| RSS取得 | FeedKit（Swift製） | 無料 |
| コンテンツ補完 | OGP抽出（og:description, og:image） | 無料 |
| AI要約・分類 | Foundation Models（iOS 26+） | 無料 |
| DB＆同期 | SwiftData + CloudKit | 無料（Apple無料枠内） |
| UI | SwiftUI | — |

ChatGPTやClaude APIを使えば高品質な要約は簡単に作れますが、APIキーの管理、サーバーの運用、従量課金と、個人開発者にとっては面倒ごとが増えます。Apple Intelligenceなら、デバイス上で動く＝ユーザーのデータが外に出ない＝プライバシーポリシーもシンプル、という三重の利点があります。

## Foundation Modelsフレームワークの使い方

iOS 26で追加されたFoundation Modelsフレームワークを使って、2つのAI処理をしています。

### 1. 記事の要約 — 自由文生成

記事の要約は、構造化出力（Guided Generation）を使わず、自由文として生成しています。要約はマークダウン形式で表示するため、太字や箇条書きを自然に含められる自由文の方が適しています。

```swift
let session = LanguageModelSession()

let prompt = """
あなたはニュース記者です。以下の記事を書き直してください。
元の記事の内容を忠実に、しかし簡潔に伝える記事を書いてください。

\(article.textForAI)

書き方:
- あなた自身が記者としてこのニュースを伝えてください。
  「この記事では」「紹介しています」のような第三者的な表現は使わないでください
- 重要な情報は省略せず、読者がこれだけで内容を理解できるように
- 複数の論点がある場合は「・」で列挙
- 核心となる事実や結論は **太字** で囲む
- 前置きは不要。本文だけを出力
"""

let response = try await session.respond(to: prompt)
```

ポイントは「ニュース記者として書き直す」という指示です。「要約してください」だと「この記事は〇〇について紹介しています」のような他人事の文章になりがちですが、記者として書くと「〇〇が発表された」「**〇〇が△△に対応**」のように、情報が直接伝わる文体になります。

### 2. カテゴリ分類 — 構造化出力（Guided Generation）

カテゴリ分類とキーワード抽出には、`@Generable`と`@Guide`を使った構造化出力を使っています。

```swift
@Generable
struct CategoryResult {
    @Guide(description: "カテゴリのアルファベット1文字")
    var category: String
    @Guide(description: "記事の重要キーワード（日本語、最大5つ）")
    var keywords: [String]
    @Guide(description: "keywordsの英訳（同じ順序、各1単語の英語）")
    var keywordsEn: [String]
}
```

`@Generable`をつけた構造体を定義し、`session.respond(to:generating:)`に渡すだけで、AIが構造化されたデータを返してくれます。JSONパースのエラーハンドリングが不要になるのは大きいです。

```swift
let response = try await session.respond(
    to: prompt, generating: CategoryResult.self
)
let result = response.content
// result.category → "A"（= テクノロジー）
// result.keywords → ["生成AI", "Apple", "WWDC"]
// result.keywordsEn → ["generativeAI", "Apple", "WWDC"]
```

カテゴリはユーザーが自由に編集できるリストで、プロンプトにはアルファベットのラベルをつけて渡しています。

```
カテゴリ一覧:
A. 政治
B. 経済
C. 社会
D. テクノロジー
...
```

AIに「Dを選べ」ではなく「最も適切なアルファベットを1つ」と指示して、結果をコード側でカテゴリ名に変換します。こうすることで、ユーザーがカテゴリを追加・変更しても、プロンプトの構造を変えずに対応できます。

### キーワードの英訳を同時に取得する理由

`keywordsEn`（英訳キーワード）を同時に生成しているのは、パーソナライズのためです。次のセクションで詳しく書きますが、iOSの`NaturalLanguage`フレームワークに内蔵されている英語のWord Embeddingを使って、キーワード同士の意味的な近さを計算しています。日本語のWord Embeddingは精度が限られるため、英訳を経由することでセマンティック検索の精度を上げています。

### AI非対応デバイスへのフォールバック

Foundation Modelsが使えないデバイス（iPhone 14以前やIntel Mac）では、AI機能を無効化してフォールバックします。

```swift
var isAvailable: Bool {
    #if canImport(FoundationModels)
    return true
    #else
    return false
    #endif
}
```

要約の代わりにOGPの説明文を表示し、カテゴリはソースのデフォルトカテゴリを割り当てます。AI無しでも「RSSリーダー + OGP付きカード表示」として普通に使えます。

## パーソナライズ — 3層のスコアリング

Seektheaの「おすすめ」タブでは、記事を関連度順に並べています。スコアリングは3つの軸の重み付き合算です。

### スコアの構成

```
最終スコア = キーワード(30%) + セマンティック(40%) + カテゴリ読了率(30%)
```

**1. キーワードベース（30%）**

ユーザーが設定した興味トピックと、読了・お気に入り記事から自動学習したキーワードを使います。記事のタイトルやキーワードに含まれるかを文字列マッチで判定します。シンプルですが、「AI」「Swift」など明確なトピックには効果的です。

**2. セマンティック（40%）**

ここがApple Intelligenceと組み合わせた面白い部分です。`NaturalLanguage`フレームワークのWord Embeddingを使って、キーワード同士の**意味的な距離**を計算します。

```swift
let wordEmbedding = NLEmbedding.wordEmbedding(for: .english)
let distance = wordEmbedding.distance(
    between: "programming", and: "software"
)
// → 近い値が返る
```

例えば、ユーザーが「プログラミング」に興味があるなら、「ソフトウェア」「開発ツール」「コンパイラ」といった直接マッチしないキーワードの記事も高スコアになります。

先ほどAIでキーワードの英訳を同時生成していたのは、このWord Embeddingが英語で最も精度が高いためです。日本語キーワード → AIで英訳 → 英語Word Embeddingで類似度計算、という流れです。

**3. カテゴリ読了率（30%）**

ユーザーがどのカテゴリの記事をよく読むかを自動で学習します。テクノロジーの記事を多く読む人には、テクノロジーの新着記事が上位に来ます。3件以上の記事があるカテゴリのみ計算するので、たまたま1件読んだだけのカテゴリが過大評価されることはありません。

### 新着ボーナス

最後に、24時間以内の記事には1.2倍のボーナスをかけます。古い記事が上位を占め続けないようにするためです。

この仕組み全体がオンデバイスで完結しているのがポイントです。ユーザーの閲覧履歴やお気に入りのデータは一切外部に送信されません。

## まとめ

Seektheaは、iOS 26のFoundation Modelsフレームワークを使って、記事の要約・分類・パーソナライズをすべてオンデバイスで実現したアプリです。

開発してみて感じたのは、オンデバイスAIの可能性の広さです。GPT-4やClaudeと比べれば性能は限られますが、「APIキー不要」「通信不要」「プライバシー完全」という利点は大きい。特に個人開発では、ランニングコストがゼロで済むのは正義です。

Foundation Modelsの`@Generable`による構造化出力は使い勝手がよく、「テキストを受け取って構造化データを返す」タイプの処理には十分実用的です。今後、対応デバイスが広がればさらに活用シーンは増えるはずです。

Seektheaは無料で公開しています。興味のある方はぜひ試してみてください。
